rtc.io faq

This is a list of questions that have been asked about rtc.io via twitter. We've attempted to provide answers here, and also working examples that can be run in requirebin.

Remember: if the sample demonstrates connectivity then you will need to open two browser tabs to run the sample (or even better - two browser instances, maybe even one using a fake media stream).

@silviapfeiffer Hi Silvia, playing w/ rtc.io, I was wondering how to switch from video+audio to audio-only w/out restarting the connection?

— ju (@delphiki) September 26, 2014

Option 1

Capture and negotiate on a video stream, then disable the video track in the captured stream.

Option 2

Capture and negotiate on a video stream, then capture an audio only stream and replace the original stream with the audio only stream.

replace-captured-media

NOTE

Both the solutions above require features of WebRTC or media streams that are currently only implemented in Chrome.
